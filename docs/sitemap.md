# サイトマップ & ルーティング設計

Next.js (App Router) では、`app` フォルダ内のディレクトリ構造がそのまま URL になります。
ここでは、作成するページと対応するファイルパスを定義します。

## ページ構成一覧

| ページ名         | URL パス      | ファイルパス (予定)       | 説明                                        | 学習ポイント                             |
| ---------------- | ------------- | ------------------------- | ------------------------------------------- | ---------------------------------------- |
| **トップページ** | `/`           | `app/page.tsx`            | サイトの顔。ヒーローセクションなどを表示。  | 基本的なページの作り方                   |
| **プロフィール** | `/about`      | `app/about/page.tsx`      | 自己紹介詳細。経歴などを記載。              | 新規ルート（フォルダ）の作成             |
| **実績一覧**     | `/works`      | `app/works/page.tsx`      | 制作実績の一覧を表示。                      | 配列データのマップ処理                   |
| **実績詳細**     | `/works/[id]` | `app/works/[id]/page.tsx` | 各実績の詳細。`[id]` の部分は動的に変わる。 | **動的ルーティング (Dynamic Routes)**    |
| **お問い合わせ** | `/contact`    | `app/contact/page.tsx`    | 連絡フォームなど。                          | フォーム処理、クライアントコンポーネント |

## 共通レイアウト構造

Next.js では `layout.tsx` というファイルが、その階層以下のページすべてに共通して適用されます。

- **ルートレイアウト** (`app/layout.tsx`)
  - 全ページ共通のデザイン（ヘッダー、フッター、HTML の骨組み）を定義します。
  - ここにナビゲーションバーを置くことで、ページ遷移してもヘッダーが維持されます。

## ディレクトリツリー予定図

```text
app/
├── layout.tsx      # 共通レイアウト (Header, Footer)
├── page.tsx        # トップページ (/)
├── globals.css     # 全体スタイル
├── about/
│   └── page.tsx    # プロフィール (/about)
├── works/
│   ├── page.tsx    # 実績一覧 (/works)
│   └── [id]/
│       └── page.tsx # 実績詳細 (/works/1, /works/project-a 等)
└── contact/
    └── page.tsx    # お問い合わせ (/contact)
```
